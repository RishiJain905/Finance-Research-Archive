```json
{

  "url": "https://www.federalreserve.gov/newsevents/pressreleases.htm",
  "allowed_prefixes": ["https://www.federalreserve.gov/newsevents/pressreleases/"],
  "url_blocklist_fragments": ["/about/", "/careers/", "/education/"],
  "max_links": 3,
  "required_keywords": ["monetary policy", "interest rates", "inflation"],
  "blocked_keywords": ["job opening", "application", "salary"],
  "min_word_count": 120,
  "enabled": true,
  "expected_language": "en",
  "allowed_page_types": ["article", "press_release", "speech"]
}
```

**Seed site format** (from `config/seed_sites.json`):
```json
{
  "id": "federalreserve.gov",
  "enabled": true,
  "domain": "federalreserve.gov",
  "start_urls": ["https://www.federalreserve.gov/newsevents/pressreleases.htm"],
  "allowed_prefixes": ["https://www.federalreserve.gov/"],
  "blocked_fragments": ["/about/", "/careers/", "/education/"],
  "max_depth": 2,
  "max_pages": 30,
  "topic": "macro catalysts"
}
```

**Common patterns across ALL configs:**
- `enabled` (boolean, defaults true) — gate for whether source runs
- `disabled_reason` (string, optional) — explanation when disabled
- `url_blocklist_fragments` / `blocked_fragments` — case-insensitive substring URL exclusion
- `max_links` / `max_pages` — volume control
- `required_keywords` / `blocked_keywords` — content filtering
- `topic` — always `"macro catalysts"` or `"market structure"`

### Task C1: Config Validation Script

**File:** `scripts/validate_source_expansion.py`
**Purpose:** Validate all expansion config files for structural integrity, required fields, valid enums, and cross-references.

**Functions:**
- `load_json(path: Path) -> dict` — Load JSON with error handling
- `validate_families_config(config: dict) -> list[str]` — Validate article_source_families.json
- `validate_batch_config(config: dict, families: dict) -> list[str]` — Validate a batch config
- `validate_target(target: dict, families: dict, target_classes: dict) -> list[str]` — Validate a single target
- `validate_all_configs(base_dir: Path) -> dict[str, list[str]]` — Validate all configs
- `main()` — CLI entry point with summary output

**Validation rules:**
1. All required fields present in each target
2. `family` references a valid family in families config
3. `target_class` references a valid class in families config
4. `priority_tier` is one of S/A/B/C
5. `enabled` is boolean
6. `url` is valid URL format
7. `allowed_prefixes` is non-empty array
8. `max_links` is positive integer
9. `min_word_count` is positive integer
10. No duplicate URLs across all batches
11. Disabled targets have `disabled_reason`
12. Family target count matches actual count in batches
