# V2.7 Part 5 — Source Performance Analytics and Adaptive Control Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a source analytics system that tracks per-source performance metrics and generates actionable recommendations for source management.

**Architecture:** Two-script system: `source_analytics.py` reads all record directories (accepted, review_queue, rejected, filtered_out) to compute per-source statistics, saves stats files to `data/source_analytics/`. `source_recommendations.py` reads stats files and generates recommendations based on configurable threshold rules, saves to `data/source_recommendations/`. Phase 1 is read-only recommendations — no auto-editing configs.

**Tech Stack:** Python 3.11+, pathlib, json, datetime, unittest/pytest

## Context for Implementer

### Codebase Patterns to Follow

**BASE_DIR pattern (ALL scripts use this):**
```python
from pathlib import Path
BASE_DIR = Path(__file__).resolve().parent.parent
```

**load_json/save_json pattern:**
```python
def load_json(path: Path, default=None):
    if not path.exists():
        return default
    with path.open("r", encoding="utf-8") as f:
        return json.load(f)

def save_json(path: Path, data) -> None:
    path.parent.mkdir(parents=True, exist_ok=True)
    with path.open("w", encoding="utf-8") as f:
        json.dump(data, f, indent=2, ensure_ascii=False)
```

**Source metadata in JSON records:**
```json
"source": {
    "name": "Federal Reserve Bank of New York",
    "url": "https://www.newyorkfed.org/research/staff_reports/sr829.html",
    "domain": "www.newyorkfed.org",
    "published_at": "2017-11-01",
    "source_type": "research_paper"
}
```

**Filtered-out TXT record format (parse headers):**
```
TARGET: New York Fed Liberty Street Economics
URL: https://libertystreeteconomics.newyorkfed.org/...
INDEX_URL: https://libertystreeteconomics.newyorkfed.org/
```
Extract source from `TARGET:` line, domain from `URL:` line.

### Task B1: Source Analytics — Record Loading Functions

**File:** `scripts/source_analytics.py`
**Functions:**
- `load_json_records(directory: Path) -> list[dict]` — Load all JSON files from a directory
- `load_filtered_records(directory: Path) -> list[dict]` — Parse TXT files with header metadata
- `extract_source_from_record(record: dict) -> dict` — Extract source info from JSON record
- `extract_source_from_filtered(filtered: dict) -> dict` — Extract source info from filtered TXT record

**Key implementation details:**
- JSON records have `source` dict with `name`, `domain`, `url`, `source_type`
- Filtered TXT records have headers: `TARGET:`, `URL:`, `INDEX_URL:`
- Parse filtered headers by reading lines until blank line, splitting on `: `
- Domain extraction: parse from URL using urllib.parse or string splitting

### Task B2: Source Analytics — Stats Computation

**Functions:**
- `compute_source_stats(accepted, review, rejected, filtered) -> dict[str, dict]` — Main stats computation
- `compute_ratios(counts: dict) -> dict` — Calculate accepted/review/rejected/filtered ratios
- `compute_avg_priority_score(records: list[dict]) -> float` — Average triage priority score
- `compute_avg_verification_confidence(records: list[dict]) -> float` — Average llm_review.verification_confidence
- `get_last_seen_at(records: list[dict]) -> str` — Most recent created_at timestamp

**Key implementation details:**
- Group records by `source.domain` (primary key)
- For each source, count records in each bucket (accepted/review/rejected/filtered)
- Calculate ratios: count / total_records_seen
- Priority scores: may come from triage metadata or computed scores in records
- Verification confidence: `record.get("llm_review", {}).get("verification_confidence", 0)`
- Handle missing fields gracefully with defaults

**Key implementation details:**

### Task C2: Source Recommendations — Recommendation Generation

**Functions:**
- `generate_recommendation(stats: dict, rules: dict) -> dict` — Generate single recommendation

- `build_reasons(action: str, stats: dict, rules: dict) -> list[str]` — Build human-readable reasons

**Reason templates:**
- Disable: "Accepted ratio {ratio:.1%} is below {threshold:.1%} threshold", "Filtered ratio {ratio:.1%} exceeds {threshold:.1%} threshold"
- Tighten: "Review ratio {ratio:.1%} is high", "Source still produces accepted records ({count})"
- Lower max links: "Source is useful but high-volume and noisy", "Consider reducing max_links_per_run"
- Investigate: "Volatile performance with {review_ratio:.1%} review ratio", "Mixed results unclear"
- Keep: "Strong accepted ratio ({ratio:.1%})", "Reasonable filtered and review ratios"

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Filtered-out TXT format parsing fails | Robust header parsing with fallback, skip malformed files |
| Missing fields in records cause crashes | Defensive defaults for all optional fields |
| Empty directories produce no stats | Handle gracefully, produce empty stats dict |
| Recommendation rules too aggressive | Start conservative, make rules configurable |
| Performance with many records | Use dict lookups, process in single pass |
| Stats not deterministic | Use consistent sorting, deterministic filenames |
